The idea

Curated Connections is a $49/mo community matchmaking SaaS — automated 1-on-1 matching, member directories, accountability challenges, group events. It runs on Rails + Turbo/Stimulus. The kind of product that benefits enormously from an always-on AI that can handle the repetitive stuff.

The experiment: take a spare MacBook Air M1, set it up as a server, install Claude Code with a Max subscription ($200/mo), and create an AI employee that handles routine maintenance — monitoring errors, responding to support, running database tasks, checking analytics, and eventually doing things that directly grow revenue.

Why Claude Code and not the API? Anthropic's ToS prohibit using a Max subscription with third-party tools like OpenClaw. But Claude Code is Anthropic's own CLI — so running it via a Slack bot is fully compliant. You get Claude Opus at a flat $200/mo instead of unpredictable API bills.

Concepts explained

For when either of you wants to understand what's actually going on. Click to expand.

What is it? SSH lets you open a terminal on another computer. You type commands on your machine, they execute on the MacBook Air. Text only — no desktop, no mouse. Like a phone call to the machine's command line.

Do we need it here? Since the MacBook Air is physically with you, SSH is optional — you could just open the lid. But it's handy for quick checks without disturbing the setup, and essential if Piyush needs access from his machine.
What is it? macOS's built-in system for keeping programs alive. You write a small config file that says "run this script, keep it alive forever, restart it if it crashes, start it when the computer boots." Apple uses it for its own services.

Why do we need it? This is what makes the Slack bot truly always-on. Bot crashes at 3am? launchd restarts it in seconds. MacBook reboots after a power outage? launchd starts the bot automatically. No human intervention.
What is it? Normally, Slack bots need a public URL. Socket Mode flips this — our bot reaches out to Slack and holds the connection open. No public URL, no firewall config.

Why do we need it? The MacBook Air is on a home network with no public address. Socket Mode means the bot works perfectly — it only makes outbound connections, which always work regardless of your router setup.
What is it? Anthropic's claude command-line tool. When the Slack bot receives a message, it runs claude -p "your message" — giving Claude access to your codebase, CLAUDE.md context, and any tools you configure. Powered by the $200/month Max subscription — flat rate, no per-message costs.

Why not the API? The CLI inherits everything — your project context, installed tools, plugins. The API would require rebuilding all of that. Plus, Max is flat-rate vs unpredictable API billing.
What is it? A markdown file in your project root that Claude Code reads automatically. It's like an employee handbook — you write the business context, codebase conventions, deployment process, common tasks, and any standing instructions. Every time Claude responds, it has this context.

For Curated Connections: this would include your Rails architecture, database schema highlights, deployment process (Render? Heroku?), common maintenance tasks, how to check analytics, support response templates, and anything else your AI employee needs to know to be effective.
Phase 1 Nityesh

MacBook Air setup

Factory reset, configure for always-on use. ~30 minutes.

1. Factory reset

  • Back up anything you want to keep
  • Go to System Settings → General → Transfer or Reset → Erase All Content and Settings
  • Go through the macOS setup wizard — create a user account (e.g., cc or nityesh)
  • Sign in with an Apple ID (needed for App Store)
  • When asked about FileVault: skip it / turn it off
    If the MacBook restarts after a power loss, FileVault shows an unlock screen. Since it might restart unattended, skip encryption so it boots straight to the desktop.

2. Configure for always-on use

Open System Settings:

Energy / Battery
  • Prevent automatic sleeping when the display is off — ON
  • Wake for network access — ON
  • Optimized Battery Charging — ON (protects the battery while plugged in 24/7)
General → Sharing
  • Remote Login (SSH) — ON (so Piyush can connect, and for quick admin from your main machine)
  • Screen Sharing — ON (full desktop access from another Mac)
Users & Groups
  • Automatic login — set to your user account
Lock Screen
  • Require password after screen saver — set to a long delay or "Never" (so screen sharing works without auth prompts)

3. Keep it awake with the lid closed

The MacBook Air will run as a server with the lid closed (clamshell mode). Open Terminal and run:

# Prevent sleep entirely — even with the lid closed
sudo pmset -a disablesleep 1

# Verify the setting
pmset -g

# To reverse later if needed:
# sudo pmset -a disablesleep 0
Battery note: Running plugged in 24/7 can cause battery swelling over months/years. Keep "Optimized Battery Charging" enabled. Worst case, a battery replacement is $100-150. Acceptable trade-off for an always-on AI employee.

4. Physical setup

  • Plug into power (always)
  • Connect to ethernet if possible (more reliable than WiFi) — a USB-C to Ethernet adapter works
  • Close the lid, tuck it somewhere out of the way
  • Optional: plug into a power strip with a switch for easy power cycling
Phase 2 Nityesh

Software setup

Install everything the AI employee needs. ~20 minutes.

1. Install core tools

Open Terminal on the MacBook Air (before closing the lid, or SSH in from your main machine):

# Install Homebrew (package manager)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install essentials
brew install git python3 tmux

# Install Node.js (needed for Claude Code)
brew install node

# Install Claude Code
npm install -g @anthropic-ai/claude-code

# Sign in to Claude Code with your Max subscription
claude
# Follow the login prompts

2. Clone your codebase

# Set up your workspace
mkdir -p ~/Projects
cd ~/Projects

# Clone the Curated Connections repo
git clone <your-repo-url> curated-connections
cd curated-connections

# Install Ruby/Rails dependencies if you want the AI to run rails console, etc.
bundle install

3. Set up the Slack bot

Create a new Slack app for your CC workspace:

a. Go to api.slack.com/apps → Create New App → "From scratch"

b. Name it something like "CC Bot" or "CC Employee" — whatever feels right

c. Settings → Socket Mode → Enable → create an App-Level Token with connections:write. Save the xapp-... token.

d. OAuth & Permissions → Add Bot Token Scopes: app_mentions:read, chat:write, im:history, im:read, im:write, files:read

e. Event Subscriptions → Subscribe to: app_mention, message.im

f. Install to workspace → save the Bot Token (xoxb-...)

4. Install and configure the bot code

# Create the bot directory
mkdir -p ~/Projects/cc-slack-bot
cd ~/Projects/cc-slack-bot

# Set up Python environment
python3 -m venv venv
source venv/bin/activate
pip install slack-bolt slack-sdk python-dotenv websocket-client

# Create .env file
cat <<'EOF' > .env
SLACK_BOT_TOKEN=xoxb-your-bot-token
SLACK_APP_TOKEN=xapp-your-app-level-token
AUTHORIZED_USERS=UNITYESH,UPIYUSH
PROJECT_DIR=/Users/cc/Projects/curated-connections
CLAUDE_TIMEOUT=1800
EOF

# Copy the bot script (from the Every repo or write a fresh one)
# The same claudie_bot.py pattern works — just change the config

5. Test it

python claudie_bot.py

# You should see:
Claudie starting — authorized users: {'UNITYESH', 'UPIYUSH'}

DM the bot in Slack. If it responds — you're in business.

Phase 3 Nityesh

Make it always on

Survive crashes, reboots, and power outages.

1. Create a launchd service

cat <<'EOF' > ~/Library/LaunchAgents/com.cc.slack-bot.plist
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
"http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
<key>Label</key>
<string>com.cc.slack-bot</string>
<key>ProgramArguments</key>
<array>
<string>/Users/cc/Projects/cc-slack-bot/venv/bin/python</string>
<string>/Users/cc/Projects/cc-slack-bot/claudie_bot.py</string>
</array>
<key>WorkingDirectory</key>
<string>/Users/cc/Projects/cc-slack-bot</string>
<key>RunAtLoad</key><true/>
<key>KeepAlive</key><true/>
<key>StandardOutPath</key>
<string>/Users/cc/Projects/cc-slack-bot/bot.log</string>
<key>StandardErrorPath</key>
<string>/Users/cc/Projects/cc-slack-bot/bot-stderr.log</string>
</dict>
</plist>
EOF

2. Load, verify, and test

# Start the service
launchctl load ~/Library/LaunchAgents/com.cc.slack-bot.plist

# Verify it's running
launchctl list | grep cc

# Test crash recovery — kill it, watch it come back
kill $(pgrep -f claudie_bot)
sleep 3 && pgrep -f claudie_bot

# Test reboot survival
sudo reboot
# Wait 2-3 min, DM the bot in Slack
It's alive! Your AI employee is now running 24/7 on the MacBook Air. Close the lid, tuck it away. It'll keep working.
Phase 4 Piyush

Piyush's access

How Piyush uses the AI employee day-to-day.

Hey Piyush!

Nityesh handles all the setup. For you, it's simple: you have a new teammate in Slack. DM it, @mention it in channels, ask it anything about Curated Connections — it has access to the codebase, can check things, and respond intelligently.

How to use it

  • DM the bot directly — just like messaging any teammate
  • @mention it in channels — it responds in a thread
  • Thread replies continue the conversation — it remembers context within a thread
  • Attach screenshots or files — it can read images and documents

Example things to ask

"How many new signups did we get this week?" — it can check the database

"There's a bug in the matching algorithm — user X got matched with themselves" — it can investigate the code

"Draft a response to this support email" — paste the email, it writes a reply

"What's our MRR trend looking like?" — it can run analytics queries

"Can you add a 'last active' column to the member directory?" — it can write and test code

What it can't do (yet)

  • Deploy to production (we'll gate this intentionally at first)
  • Access external services it hasn't been configured for
  • Replace human judgment on product decisions — it's a tool, not a cofounder

Admin commands

Quick reference for managing the MacBook Air server.

# Check bot status
launchctl list | grep cc
pgrep -f claudie_bot

# View live logs
tail -f ~/Projects/cc-slack-bot/bot.log

# Restart the bot
launchctl unload ~/Library/LaunchAgents/com.cc.slack-bot.plist
launchctl load ~/Library/LaunchAgents/com.cc.slack-bot.plist

# Pull latest code
cd ~/Projects/curated-connections && git pull

# Interactive Claude Code session
tmux new -s claude
cd ~/Projects/curated-connections && claude
# Detach: Ctrl+B then D | Reattach: tmux attach -t claude

# Re-enable sleep (if you want to use the MacBook normally again)
sudo pmset -a disablesleep 0
