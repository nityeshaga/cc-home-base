Concepts explained

For when either of you wants to understand what's actually going on. Click to expand.

What is it? SSH lets you open a terminal on another computer over the network. You type commands on your main machine, they execute on the MacBook Air. Text only — no desktop, no mouse.

Do we need it? Yes — for quick admin from your main machines without walking over, and essential when you're travelling. Combined with Tailscale, you can SSH into the MacBook Air from a coffee shop, a co-working space, or another city.
What is it? Tailscale creates an invisible, encrypted tunnel between your devices. Your laptops and the MacBook Air join the same private network — as if they were on the same WiFi, even when you're in different cities. Each device gets a stable address (like 100.x.x.x) that never changes.

Why do we need it? At home, you can SSH using the local network. But the moment you step out — a coffee shop, a co-working space, travelling — your home network is unreachable. Tailscale makes the MacBook Air accessible from anywhere in the world, securely, without exposing anything to the public internet. Free for personal use.

Analogy: If SSH is a phone call, Tailscale is the phone network. SSH needs a way to reach the MacBook Air — at home that's your WiFi, but outside the house, Tailscale provides that path.
What is it? macOS's built-in system for keeping programs alive. You write a small config file that says "run this script, keep it alive forever, restart it if it crashes, start it when the computer boots." Apple uses it for its own services.

Why do we need it? This is what makes the Slack bot truly always-on. Bot crashes at 3am? launchd restarts it in seconds. MacBook reboots after a power blip? launchd starts the bot automatically. No human intervention.
What is it? Normally, Slack bots need a public URL. Socket Mode flips this — our bot reaches out to Slack and holds the connection open. No public URL, no firewall config, no router setup.

Why do we need it? The MacBook Air is on your home WiFi with no public address. Socket Mode means the bot works perfectly — it only makes outbound connections, which always work regardless of your network.
What is it? Anthropic's claude command-line tool. When the Slack bot receives a message, it runs claude -p "your message" — giving Claude access to your codebase, CLAUDE.md context, and any tools you configure. Powered by the $200/month Max subscription — flat rate, no per-message costs.

Why not the API? The CLI inherits everything — your project context, installed tools, plugins. The API would require rebuilding all of that. Plus, Max is flat-rate vs unpredictable API billing.
What is it? A markdown file in your project root that Claude Code reads automatically. It's like an employee handbook — you write the business context, codebase conventions, deployment process, common tasks, and standing instructions. Every time Claude responds, it has this context.

For Curated Connections: this would include your Rails architecture, database schema highlights, deployment process, common maintenance tasks, how to check analytics, support response templates, and anything else your AI employee needs to know to be effective.
Phase 1 Piyush

Hardware setup

Factory reset and configure the MacBook Air for always-on use. ~30 minutes.

Hey Piyush!

You're setting up the MacBook Air M1 as a dedicated server. Factory reset it, configure it to stay awake with the lid closed, and get the basic software installed. After this, both of you manage it together — but this first phase is your show.

1. Factory reset

  • Back up anything worth keeping from the old install
  • Go to System Settings → General → Transfer or Reset → Erase All Content and Settings
  • Go through the macOS setup wizard — create a user account (e.g., cc)
  • Sign in with an Apple ID (needed for App Store later)
  • When asked about FileVault (disk encryption): skip it / turn it off
    If the MacBook restarts after a power loss, FileVault shows an unlock screen that needs a keyboard. Since it might restart unattended (lid closed), skip encryption so it boots straight to the desktop.

2. System Settings checklist

Open System Settings and configure each:

Energy / Battery
  • Prevent automatic sleeping when the display is off — ON
  • Wake for network access — ON
  • Optimized Battery Charging — ON (protects battery while plugged in 24/7)
General → Sharing
  • Remote Login (SSH) — ON (so you can both manage it from your main machines without opening the lid)
  • Screen Sharing — ON (full desktop access from another Mac on the same network)
Users & Groups
  • Automatic login — set to the user account you created
Lock Screen
  • Require password after screen saver — Never (so Screen Sharing works without auth prompts)

3. Install Tailscale on all three machines

Tailscale lets you both access the MacBook Air from anywhere — home, a coffee shop, travelling. Install it on all three machines:

  • Open the Mac App Store on the MacBook Air, search for Tailscale, install it
  • Open Tailscale, sign in with a shared account (Google, GitHub, or Apple ID)
  • Do the same on Nityesh's MacBook and Piyush's machine — same account
  • All three devices now appear in your Tailscale dashboard with stable IPs like 100.x.x.x
  • In Tailscale preferences on the MacBook Air, enable "Start Tailscale when I log in"
At home you can SSH using the local address (cc@macbook-air.local). Outside use the Tailscale IP (cc@100.x.x.x). Both always work.

4. Keep it awake with the lid closed

Open Terminal and run:

# Prevent sleep entirely — even with the lid closed
sudo pmset -a disablesleep 1

# Verify it worked
pmset -g

# To reverse later: sudo pmset -a disablesleep 0
Battery note: Running plugged in 24/7 can cause battery swelling over months/years. "Optimized Battery Charging" helps. Worst case, battery replacement is ~$100-150. Acceptable trade-off for an always-on AI employee.

5. Install core tools

# Install Homebrew (package manager — makes installing everything else easy)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install essentials
brew install git python3 tmux node

# Install Claude Code
npm install -g @anthropic-ai/claude-code

# Sign in to Claude Code (Nityesh's Max subscription)

# Follow the login prompts — it opens a URL you paste in any browser

6. Set it up physically

  • Plug into power (always — use a reliable outlet)
  • Connect to your home WiFi (or ethernet via USB-C adapter if you have one — more reliable)
  • Close the lid, tuck it somewhere out of the way — a shelf, behind a monitor, wherever
  • Note the local IP address or hostname (shown in System Settings → Sharing, something like cc@macbook-air.local)
Hardware done! The MacBook Air is now a headless server. Both of you can SSH in from your main machines, or just walk over and open the lid. Time to set up the Slack bot together.
Phase 2 Nityesh + Piyush

Slack bot setup

Create the Slack app, install the bot, wire it to Claude Code. ~20 minutes.

1. Create the Slack app

Either of you can do this — whoever has admin access to your Slack workspace:

a. Go to api.slack.com/apps → Create New App → "From scratch"

b. Name it — "CC Bot", "CC Employee", or whatever feels right

c. Settings → Socket Mode → Enable → create an App-Level Token with connections:write. Save the xapp-... token.

d. OAuth & Permissions → Add Bot Token Scopes: app_mentions:read, chat:write, im:history, im:read, im:write, files:read

e. Event Subscriptions → Subscribe to bot events: app_mention, message.im

f. Install to workspace → save the Bot Token (xoxb-...)

g. Find both your Slack user IDs: Profile → ⋮ menu → "Copy member ID"

2. Install the bot on the MacBook Air

SSH in from either of your machines, or open the lid:

# SSH from your main machine (same WiFi network)
ssh cc@macbook-air.local

# Create the bot directory
mkdir -p ~/Projects/cc-slack-bot
cd ~/Projects/cc-slack-bot

# Set up Python environment
python3 -m venv venv
source venv/bin/activate
pip install slack-bolt slack-sdk python-dotenv websocket-client

# Create .env file with your tokens
cat <<'EOF' > .env
SLACK_BOT_TOKEN=xoxb-your-bot-token
SLACK_APP_TOKEN=xapp-your-app-level-token
AUTHORIZED_USERS=UNITYESH_ID,UPIYUSH_ID
PROJECT_DIR=/Users/cc/Projects/curated-connections
CLAUDE_TIMEOUT=1800
EOF

3. Clone the Curated Connections codebase

cd ~/Projects
git clone <your-repo-url> curated-connections

# Optional: install Rails dependencies so the AI can use rails console
cd curated-connections
bundle install

4. Add the bot script

Copy the claudie_bot.py from the Every Consulting repo (or Nityesh will set this up). The bot script handles:

  • Slack Socket Mode connection (no public URL needed)
  • Auth gating — only your two Slack IDs can interact
  • Thread-based conversations with Claude session continuity
  • Markdown to Slack formatting conversion
  • File/image attachment handling
  • Audit logging of every interaction

5. Test it

cd ~/Projects/cc-slack-bot
source venv/bin/activate
python claudie_bot.py

# You should see:
Claudie starting — authorized users: {'UNITYESH', 'UPIYUSH'}

DM the bot in Slack. If it responds — you're in business.

Phase 3 Nityesh + Piyush

Make it always on

Survive crashes, reboots, and power outages — permanently.

1. Create a launchd service

This tells macOS to keep the bot alive forever:

cat <<'EOF' > ~/Library/LaunchAgents/com.cc.slack-bot.plist
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
"http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
<key>Label</key>
<string>com.cc.slack-bot</string>
<key>ProgramArguments</key>
<array>
<string>/Users/cc/Projects/cc-slack-bot/venv/bin/python</string>
<string>/Users/cc/Projects/cc-slack-bot/claudie_bot.py</string>
</array>
<key>WorkingDirectory</key>
<string>/Users/cc/Projects/cc-slack-bot</string>
<key>RunAtLoad</key><true/>
<key>KeepAlive</key><true/>
<key>StandardOutPath</key>
<string>/Users/cc/Projects/cc-slack-bot/bot.log</string>
<key>StandardErrorPath</key>
<string>/Users/cc/Projects/cc-slack-bot/bot-stderr.log</string>
</dict>
</plist>
EOF

2. Load, verify, and test

# Start the service
launchctl load ~/Library/LaunchAgents/com.cc.slack-bot.plist

# Verify it's running
launchctl list | grep cc

# Test crash recovery — kill it, watch launchd bring it back
kill $(pgrep -f claudie_bot)
sleep 3 && pgrep -f claudie_bot

# Test reboot survival
sudo reboot
# Wait 2-3 min, then DM the bot in Slack
It's alive! Your AI employee is running 24/7 on the MacBook Air. Close the lid, tuck it away. It'll keep working. If either of you needs to debug, SSH in or just walk over and open it.
